## 7. 오케스트레이션 운영 개선 (2026-06-23 회고)

Book Dash Scheme B 트랙(정찰 4차 → ADR Amd#4~#6 → 본체 이식 → 게이트①②③ → 1단계 실적재)을 회고해 도출한 운영 개선. §5 핵심 원칙은 유지하되, 왕복·전달 부담을 줄이는 4건을 추가한다.

| # | 개선 | 내용 |
|---|---|---|
| 1 | **문서 전용 커밋의 diff 왕복 축소** | ADR·intent·backlog 등 순수 문서 커밋은 회귀 위험 낮음 → "커밋 직후 같은 보고에 diff 첨부"로 한 번에 검토. 단 **적재용 본체 코드(sync 등)는 push 전 diff 검토 종전대로 유지**(축소 금지). |
| 2 | **정찰 작업지시서를 넓게 묶기** | 한 스텝씩 좁히면 왕복 증가(Scheme B 경로 정찰 4차 사례). 정찰 지시서에 "예상되는 다음 질문까지 한 번에 관찰" 범위 포함. 예: "WP API에 없으면 책페이지 HTML `#read-book`도 같이 떠라"를 1차에 묶기. |
| 3 | **ADR 분할 최소화** | 한 덩어리 결정은 정찰을 충분히 끝낸 뒤 1~2개 Amendment로 묶어 박제. 단 **사후 발견(육안검수 등 나중 드러난 결함)은 별도 Amendment 불가피**. |
| 4 | **작업지시서 길이 절제** | 제약·배경·예고 과다 시 핵심이 묻히고 PM 전달 부담 증가. 핵심(목적·실행·예상승인표) 우선, 배경·예고는 짧게. |
