## 3. Wave 계획

| Wave | 항목 | 성격 | 리스크 |
|---|---|---|---|
| **Wave 1** | F1 완독 안내 · F2 다시듣기 · F3 표지 시작 유도 · F4 토글 라벨 | 컴포넌트 내부 UI 추가. 데이터·스키마 변경 0 | 낮음 — 기존 상태 재사용, 회귀면 좁음 |
| **Wave 2** | F5 뒤로가기 보호 · F6 자막 숨김 · F7 스와이프 | 라우팅·제스처 개입. 기존 뷰어(Asb·Html)와 동작 일관성 검토 필요 | 중 — 브라우저 뒤로가기 가로채기는 이탈 UX에 영향 |
| **Wave 3** | F8 탭 일시정지 · F9 한국어 해석(**ADR 선행 필수**) · F10 이미지 내 하이라이트 | 기획·법무·스키마 판단 동반 | 높음 — F9는 번역 저작권·품질·저장 위치 결정 선행 |

### F9 별도 주의
한국어 해석은 **ADR 선행이 필수**다. (a) 번역문의 저작권 지위(CC BY 원문의 2차 저작물 — ADR-0023 §2.6 어트리뷰션 승계), (b) 저장 위치(`book_text` 확장 vs 별도 테이블 — Hard Rule 8에 따라 스키마 변경 전 ADR), (c) 생성 주체(사람 검수 여부)를 정하기 전에는 착수하지 않는다.
