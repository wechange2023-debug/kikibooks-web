## §1 배경 — 왜 계획을 재정비하는가

- 그동안 파이프라인은 **자동추출 결과를 100% 신뢰**하는 전제 위에 서 있었다.
- 사람 눈 검수에서 이 전제가 깨졌다: 잔존 **word-mixing**(단어 뒤섞임)·**SFX 오분류**(효과음/장식글이 본문으로 섞임)가 확인되었다.
- 따라서 **"자동추출 = 초벌(draft)"로 격하**하고, **사람 검수를 파이프라인의 정식 단계로 승격**한다. 검수는 더 이상 선택적 사후 확인이 아니라, TTS 이전에 반드시 통과해야 하는 게이트다.

## §2 확정 파이프라인 (팀장 확정)

**추출(초벌) → 관리자 사람검수·수정 → TTS 생성 → marks/하이라이트 → 책 단위 서비스 ON**

| # | 단계 | 입력 | 출력 | 책임 주체 |
|---|---|---|---|---|
| 1 | 추출(초벌) | PDF harvest v1 (154권) | 초벌 텍스트 JSON (`out_fixed_154`, 커밋됨) | 워커 |
| 2 | 관리자 사람검수·수정 | 초벌 텍스트 + 페이지 이미지 | 확정 텍스트 + 검수 상태·검수자 | 팀장(검수 실행) / 오케(화면 설계) / 워커(구현) |
| 3 | TTS 생성 | 확정 텍스트 | mp3 + speech marks | 워커(스크립트) / 팀장(Storage·DB 쓰기) |
| 4 | marks/하이라이트 | mp3 + marks.json | 뷰어 단어 단위 하이라이트 | 워커(뷰어 구현) |
| 5 | 책 단위 서비스 ON | 위 전 단계 완료된 책 | `is_active`/노출 전환 | 팀장(SQL 직접 실행) |

- **서비스 ON은 책 단위**로 한다. 일괄 ON 금지 — 각 책이 검수·TTS·marks를 모두 통과한 뒤에만 해당 책만 켠다(ADR-0037 "사람이 공개 관리" 원칙과 정합).

## §3 범위·숫자 정의 (혼동 방지 — 반드시 분리 표기)

세 종류의 숫자를 **절대 합산하지 않는다.**

- **카탈로그 서비스 범위**: Book Dash 실측 **총 209권** = 활성 **206** + 비활성 **3** (state-audit §5). "전권 서비스"가 활성 206권 유지를 뜻하는지, 비활성 3권까지 포함한 209권을 뜻하는지는 **미확정**. ※ **팀장 확정 대기 — §11 참조.**
- **하이라이트·TTS 대상**: **신규 154권** + **기존 44권** — 이 둘은 별개 숫자다. "198권" 식으로 합산 호칭 금지.
- **기존 44권 자산 실측** (state-audit 인용):
  - 오디오 `book_audio` **574행** = kind `page` **530** + `cover` **44** (distinct book 44권)
  - `book-images` 버킷 루트 폴더 **49** (본문 보유 39 + cover-only 10)
  - `books.has_audio = true` = **44건** (전부 `book_dash`)
- **기존 44권은 이번 트랙에서 변경하지 않는다**(팀장 확정). Ruth 단일 음성 유지, 재생성·재업로드 없음.

## §5 Phase A — 데이터 모델 (가장 시급)

**recon 결과 (state-audit §3 실측)**: 검수 상태·확정 텍스트·검수자·캐릭터매핑을 담을 **DB 스키마가 전무**하다. `books`(19컬럼)·`book_audio`(11컬럼) 어디에도 검수 관련 컬럼이 0건이며, **DB에 본문 텍스트 테이블 자체가 없다** — 확정 텍스트는 현재 로컬 JSON뿐이다.

이 트랙 전체가 A에 물려 있으므로 A가 가장 시급하다.

**필요한 개념 4가지** (여기서는 **나열만** 한다. 컬럼명·DDL·상태값 확정은 **ADR-0046**에서 한다. 이 문서에서 스키마를 확정하지 않는다):

1. **책별 검수 상태** — 추출→검수중→검수완료→TTS완료→서비스ON 같은 중간 단계 추적(현재 `is_active`는 최종 ON만 표현).
2. **확정 텍스트 저장** — 검수·수정된 지문을 DB에 보관(현재 로컬 JSON뿐).
3. **검수자·검수 시각** — 누가 언제 확정했는지.
4. **캐릭터-화자 매핑 (옵션)** — 대사 화자 지정(옵션 기능, 필수 아님).
