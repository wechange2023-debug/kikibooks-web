# 정본 없는 157권에서의 추출 전략 선택 (미검증 가설 — 다음 세션 첫 안건)

> 작업정리 지시서(2026-07-10) STEP 2-2. **미검증 가설이다. 결론 없음. 실행 안 함.**

## 문제

PDF 텍스트 추출 전략 v1(객체 순서)과 v2(좌표 재조립)는 **책별로 승자가 다르다**
(대조군 43권 실측: 개선 12 / 후퇴 14 — `docs/recon/2026-07-10-harvest-gate-v2.md` §3).
대조군에서는 정본과 대조해 권별 우세 전략을 고를 수 있지만, **157권에는 정본이 없다.**
정답지 없이 어느 추출을 믿을지 결정할 방법이 필요하다.

## 가설

**두 전략의 출력이 서로 일치할수록 그 추출은 정확하다.**
v1↔v2 출력 간 일치도(예: 단어 시퀀스 편집거리 기반 유사도)를 **정본 없이 계산 가능한
신뢰도 대용치(proxy)**로 쓴다. 일치도가 높은 권은 자동 채택, 낮은 권은 검수 큐로 보낸다.

직관적 근거(아직 근거일 뿐): 두 전략은 서로 다른 실패 양식(객체 순서 무작위 vs 블록
기하 오판)을 가지므로, 둘이 같은 답을 내면 둘 다 맞았을 가능성이 높다.

## 검증 설계 (대조군 43권 — 정본 보유라 채점 가능)

1. 권별로 `agree(v1, v2)` = 두 출력 간 단어 수준 유사도(1 − 편집거리/평균 길이)를 계산.
2. 권별로 `acc_best` = max(v1 정확도, v2 정확도) — 이미 측정돼 있음(gate v2 데이터).
3. `agree` vs `acc_best`의 **상관**(스피어만 순위상관 권장 — 분포 왜도 큼)을 측정.
4. 보조 검증: `agree` 상위/하위 10권씩의 acc_best 분포 비교 (임계값 후보 도출용).

## ★ 반증 조건 (무엇이 나오면 이 가설은 틀린 것인가)

1. **스피어만 상관계수 < 0.5** → 일치도는 정확도의 대용치가 못 된다. **가설 폐기.**
2. **높은 일치·낮은 정확 사례 존재**: `agree ≥ 95%`인데 `acc_best < 90%`인 권이
   1권이라도 나오면 — 두 전략이 **같은 방식으로 함께 틀리는** 실패 양식이 실재한다는
   뜻이므로, proxy를 단독 신뢰 근거로 쓸 수 없다(폐기 또는 보조 신호로 강등).
3. **변별력 부재**: agree 값이 전권 95% 이상에 몰려 있으면(분산 없음) 임계값을 그을 수
   없어 실무 효용이 없다 — 폐기.

역방향(낮은 일치·높은 정확)은 가설을 폐기시키지 않는다 — proxy가 보수적으로 검수 큐를
키울 뿐이다(안전한 실패).

## 예상 산출물 (실행 시)

권별 {agree, acc_best} 표 + 상관계수 + 반증 조건 3종 판정 → 통과 시 157권 적용 규칙
(agree 임계값) 제안, 미통과 시 폐기 기록.
