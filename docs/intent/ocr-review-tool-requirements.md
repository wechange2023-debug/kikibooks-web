# OCR 검수 도구 요구사항 (도출만 — 구현·UI 설계 아님)

> 작업지시서 2026-07-10 (2) STEP 4 · 팀장 결정 I6. 근거 데이터 = OCR 파일럿 7권 실측
> (`scripts/ocr_pilot/out/*.ocr.json`, `docs/recon/2026-07-10-ocr-pilot-5books.md`).
> 본 문서는 요구사항 나열만 한다. 화면 설계·코드는 별도 트랙.

## 1. 검수자가 한 페이지에서 해야 하는 동작 (파일럿 실측 기반)

1. **원본 이미지와 OCR 초벌 텍스트 대조** — 페이지당 평균 9~44단어(파일럿 실측).
2. **오독 수정** — 대조군 실측 유형: 아포스트로피 오독(`Wangari's`→`Wangariss`),
   따옴표 시작 오독(`"T don't`, `"T'll lose` — lion 반복 발생), 첫 글자 오독(`Tt was`→`It was`).
3. **삽화 노이즈 삭제** — 실측: `®`, `\iQ`, `VAI`, `2 Ron! on Z`, `oS fey an Riis =a`(hugs p07),
   `al ¥`(back matter). 대조군 오류 9건 중 8건이 이 유형.
4. **라인 재결합** — OCR 텍스트는 시각적 줄 단위 `\n` 구분. 문장 흐름 기준으로 재결합/문단화
   필요(TTS 입력·하이라이트 문자열이 이 결과를 그대로 씀).
5. **empty 면 판정** — OCR 0자 면이 (a) 진짜 그림만 면인지 (b) 미검출인지 사람이 확인
   (실측: mogaus-gift p05·p06, the-window-seat p01 = 0자 — 원인 미확인).
6. **표기 규약 적용** — 기존 확정본(`out/{slug}.json`)은 curly apostrophe(`’`) 사용,
   OCR 출력은 straight/curly 혼재 → 규약 통일 필요(marks char offset의 기준 문자열).
7. **페이지 확정 저장 → 다음 페이지 이동.**

## 2. 필요한 화면 요소 (요구 수준)

- 원본 이미지(WP full-size, 실측 폭 2000px대 — 확대 가능해야 함)
- OCR 텍스트 편집 영역(수정 입력)
- **bbox 오버레이** — word 단위 사각형을 이미지 위에 표시, confidence 낮은 단어 구분 표시
  (실측 conf 분포: 정상 본문 84~96, 노이즈·장식 18~57)
- empty 면 표시 수단(아래 §3)
- 이전/다음 페이지 이동 + 책 단위 진행 상태(몇 면 확정/미확정)
- 페이지 확정(저장) 동작

## 3. empty 페이지 판정 표시

- OCR 0자(words 0건)면 **empty 후보로 기본 제안**하되, 확정은 반드시 사람이 한다
  (미검출 가능성 — 파일럿에서 0자 면 원인 미확인 사례 존재).
- 확정 저장 시 기존 스키마 관례를 따라 **면을 목록에 유지하고 `text: ""`**로 기록
  (ADR-0035 Amd#1 A4 — empty 면은 오디오 없음이 정상).
- "empty로 확정했다"는 사실과 "아직 검수 안 했다"를 구분할 수 있어야 한다(§5 상태 필드).

## 4. 154권 × 2,316페이지 규모에서의 병목 (실측 기반)

- **저신뢰 페이지의 사실상 재타이핑** — hugs p07(conf 57.3)류 장식·비정형 배치 페이지는
  초벌이 무의미해 통타이핑과 같아진다. conf 기준 저품질 페이지 선별 큐가 없으면
  검수 속도가 페이지 편차에 지배된다.
- **노이즈 삭제·라인 재결합의 반복 수작업** — 페이지마다 발생(대조군 오류의 8/9).
  일괄 규칙(예: conf<N 단어 기본 제외 제안)이 없으면 2,316면 전부에서 반복된다.
- **텍스트 수정 ↔ bbox 대응 붕괴** — 검수로 단어를 지우거나 고치면 `words[]` 배열과
  확정 텍스트의 단어 시퀀스가 어긋난다. 하이라이트(C안)는 이 대응이 필수이므로
  수정 후 재대응 규칙이 요구사항이다(방식은 미결).
- **이미지 로딩** — WP 원본 핫링크는 외부 종속·속도 불안정(파일럿 중 일시 타임아웃 실측).
  검수 세션용 로컬/캐시 전략 필요.
- **중단·재개** — 2,316면은 일괄 불가. 면 단위 저장과 진행 상태 영속화가 전제.
- **back matter 면 처리** — 파일럿 7권 전부에서 서비스 창 밖 후행 면(svc 13/14, 판권·뒤표지)
  이 목록에 딸려 온다. 검수 범위 밖으로 자동 표시할 기준 필요.

## 5. 저장 형식과 기존 `out/{slug}.json` 스키마의 관계

- 초벌: `scripts/ocr_pilot/out/{slug}.ocr.json`
  (`{slug, status: "raw_unreviewed", pages[{page, pdf_page, image_url, image_width/height,
  ocr_text, char_count, word_count, mean_conf, words[{t,x,y,w,h,conf}]}]}`).
- 확정본은 기존 TTS 파이프라인 입력 스키마 `[{page, image_url, text}]`(=`out/{slug}.json`)와
  **호환되어야 한다**(generate_tts.py·marks 생성이 이 스키마를 읽음 — 사실).
- bbox는 확정 텍스트와 별도로 보존되어야 한다(ADR-0039 D5 — 재추출 회피·C안 좌표).
  초벌 파일 유지 vs 확정 파일에 병합은 **미결**(recon 문서 판단 요청).
- 필요 상태 필드(현 스키마에 없음): 면 단위 검수 상태(미검수/확정/empty 확정),
  검수 일시·검수자, 원본 이미지 식별(image_file/pdf_page).
