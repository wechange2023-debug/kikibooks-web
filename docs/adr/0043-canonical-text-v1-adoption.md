# ADR-0043: 정본 텍스트 선택 — v1 채택 + 검증 사전등록

- 상태: Accepted (잠정). **lock은 Prong-1 기계검증 + Prong-2 사람검증 후 개정으로 확정.**
- 날짜: 2026-07-14
- 관련: ADR-0041(agree 보조신호)·ADR-0042(순회 게이트 사전등록)·
  scripts/pdf_harvest/out_154/_154_summary.json(순회 실측)

## 1. Context

- 154권 전량 순회 완료(2026-07-14): v2·v1 각 154/154 성공, 실패 0.
- agree 분포: Q1 = **67.48** (min 0 / median 81.91 / max 100, IQR 29.75).
  게이트 결과 **AUTO 115 / REVIEW 39** (MANUAL·WORDLESS-OK·LICENSE-HOLD 각 0).
- W-NOLAYER **0%** → ADR-0042 중단기준 `<15%` 구간, 자동 경로 건강.
- 정답지 없음 → 정확도 직접측정 불가. 정본은 추출본 중에서 골라야 한다.

## 2. 결정 (잠정)

**v1(pypdf 객체순 추출)을 154 코퍼스의 정본 텍스트로 채택한다.**
v2(pdfplumber 좌표 재조립)는 교차검증(agree 필터, ADR-0041)로 소임 완료 — 정본 아님.

## 3. 근거

- (a) v2 좌표기반의 **산포/디스플레이 조판 실패**(글자 낱개 분해·행 순서 뒤섞임)는
  이 코퍼스 조판에서 기지 사항(대조군 how-about-you·what-is-it 전례).
- (b) 낮은 agree의 원인은 v2의 하향이탈(단어깨짐)이었다 — 파일럿 하위 3권 육안
  전수에서 **3/3 v1 정상·v2 깨짐** 확인(a-day-out `I c a n ss ee…` vs v1 정상문).
- (c) 그림책의 단순 레이아웃(면당 1~3문장) → v1의 약점인 객체순서 무작위(읽기순서
  뒤섞임)가 발현할 복잡 조판 자체가 희소.

## 4. 검증 사전등록 (★데이터 보기 전 고정 — 사후 합리화 금지)

### Prong-1 (기계, 154 전권 — scatter_check.py)

- v1·v2 각각 "산포지표" 산출: single_char_ratio(주 판별자)·mean_alpha_token_len·
  fragment_hits. 권별 판정 = **v2_worse / v1_worse / tie**(임계값은 스크립트 상수).
- **예측**: 대다수 v2_worse 또는 tie. v1_worse는 극소수. REVIEW 39에 v2_worse 집중.
- **반증선**: v1_worse가 **전체 >15권** 또는 **REVIEW 39 중 >10권**이면
  v1 정본 전제가 흔들림 → **재검토(STOP)**.
- 부가: v1·v2 **둘 다** 고산포인 권(=함께깨짐 후보) → 사람검수 세트로.

### Prong-2 (사람, 한정표본 — 다음 세션)

- v1의 비-자동검출 위험 = **문장 순서 뒤섞임(의미적 오류)** — 산포지표로 안 잡힘.
- 대상: Prong-1 플래그 권 전부 + AUTO 무작위 표본 + REVIEW 39 표본을
  PDF 캐시 페이지와 대조. (정답지 부재 → **사람 눈이 유일 오라클**.)

## 5. Lock 규칙

- 검증 통과 권 → v1 정본 확정.
- 잔여 실패 권 → MANUAL(K6) 또는 **해당 권 한정** v2 채택.
- 확정은 본 ADR 개정(**Amd#1**)으로만. 무단 전환 금지.

## 6. 한계

- 정답지 부재 → **판본상이(edition mismatch)형 오류는 원리상 미검출**
  (PDF와 서비스 이미지가 다른 판이면 둘 다 같은 방향으로 틀림). 사람표본(Prong-2)에
  의존하며, 그 표본 밖 잔여 위험은 수용한다(스팟 검수는 운영 단계에서 계속).
