# ADR-0013 — 표지 썸네일 노출 시 어트리뷰션 정책 (랜딩·홈·라이브러리 공통)

**상태** Accepted
**날짜** 2026-05-19
**관련** `docs/adr/0012-landing-page-static.md`(phase-09a 설계 — 본 ADR과 함께 생성), `docs/guidelines/license-rules.md`(4절 어트리뷰션 4요소·5절 AttributionBox), `docs/intent/screen-01-landing.md`(인기 책 섹션), `tasks/phase-09a-screen-01-landing-static.json`, `supabase/migrations/001_initial_schema.sql`(books — title·author·illustrator·original_url·license·attribution_text), `PLAN.md` 15절(CC BY 4.0 어트리뷰션 4요소), `claude.md` 2절 Hard Rule 1·3

## 1. 배경

phase-09a 랜딩 페이지는 인기 책 6권의 **표지 이미지**를 노출한다. phase-10(홈)·phase-13(라이브러리)도 책 목록을 표지 썸네일 그리드로 보여줄 것이다.

CC BY 4.0은 저작물을 사용·전시할 때 저작자 표시를 요구한다(PLAN.md 15절: 저작자·제목·라이선스·원본 URL 4요소). **표지 이미지를 노출하는 것 자체가 저작물 사용**이므로 어트리뷰션 의무가 발생한다.

`license-rules.md` 5절은 **책 상세 페이지**의 `AttributionBox`(4요소 전체 표시)를 규정하지만, 목록 화면의 작은 표지 썸네일에 대한 정책은 없다. phase-09a 진입 전 사용자와의 합의(2026-05-19)에서 이 공백을 메우기로 했다.

사전 합의에서 사용자가 원래 정한 D2는 "표지만 노출(제목·저자 없음)"이었다. 그러나 표지만 노출하면 (1) 어트리뷰션 의무를 충족하지 못하고, (2) 비로그인 방문자는 표지 클릭이 `/signup`으로 가므로 책 상세 페이지의 `AttributionBox`에 영영 도달할 수 없다. 사용자는 이 검토 결과를 받아 **D2를 수정**해 캡션을 추가하기로 명시 승인했다(2026-05-19).

본 ADR은 목록 화면 표지 썸네일의 어트리뷰션 정책을 박제하며, phase-09a·10·11·13이 공통으로 따른다.

## 2. 결정

### 결정 1 — 표지 썸네일에는 제목·저자 캡션을 함께 노출 (D2 수정)

목록 화면(랜딩·홈·라이브러리)에서 책 표지 썸네일을 노출할 때, **표지 바로 아래에 책 제목과 저자(author)를 캡션으로 표시**한다. "표지만 노출"(D2)은 본 결정으로 폐기한다.

- 1차안은 **"제목 + 저자" 두 줄** 캡션이다.
- 캡션 두 줄이 6권 그리드(특히 모바일 390px)에서 시각적으로 과밀하면 **"제목만"으로 후퇴**한다. 이 판단은 phase-09a CP3 구현 중 실제 레이아웃을 보고 내리며, 결과를 사용자에게 보고한다.
- "제목만"으로 후퇴하더라도 결정 2의 푸터/섹션 안내가 라이선스·저작자 도달 경로를 보장한다.

**실측 보강 (phase-09a CP2, 2026-05-19).** phase-09a CP2 검증 중 DB를 실측한 결과, 활성 책 896권 중 **540권(60%)이 `author`가 NULL**이다. 플랫폼별 분포는 Book Dash 54권 누락 0%, GDL 842권 중 540권(64%) 누락 — 원천 플랫폼마다 적재 품질이 다르다. 따라서 캡션은 **제목(필수) + 저자(있을 때만)**로 동작하며, 카드마다 1줄/2줄이 섞이는 시각 변동은 정상이다. 저자가 NULL인 책은 제목 한 줄 + 푸터 CC BY 안내(결정 2)가 어트리뷰션 하한선을 지킨다. 4요소 완전 표시 의무는 책 상세 페이지가 진다(결정 3).

### 결정 2 — 표지를 노출하는 화면에는 CC BY 안내 문구를 둔다

표지 썸네일을 노출하는 화면에는 푸터 또는 섹션 하단에 다음 안내를 표시한다.

> "모든 도서는 CC BY 4.0 라이선스이며, 글·그림 저작자와 원본 출처는 각 책 상세 페이지에 표시됩니다."

이 문구는 라이선스 종류(CC BY 4.0)를 화면에서 직접 알리고, 4요소 전체(저작자·제목·라이선스·원본 URL)는 책 상세 페이지에서 확인된다는 경로를 명시한다.

### 결정 3 — 4요소 전체 표시 의무는 책 상세 페이지가 진다

목록 화면의 캡션(제목·저자) + 안내 문구는 **요약 표시**다. CC BY 4.0 4요소 전체 — 저작자·제목·라이선스(링크)·원본 URL — 의 완전한 표시 의무는 책 상세 페이지의 `AttributionBox`가 진다(`license-rules.md` 5절, phase-11). 본 ADR은 그 의무를 줄이거나 바꾸지 않는다.

### 결정 4 — 목록 화면 표지 노출은 closed environment 안에서만

표지 썸네일을 노출하는 화면이라도, 읽기 콘텐츠 경로와 마찬가지로 검색엔진 색인 정책은 화면별 성격을 따른다. 랜딩(`/`)은 마케팅 페이지라 색인을 허용하지만, 홈·라이브러리(`/home`·`/library`)는 `robots.txt`로 색인을 차단한다(PLAN.md 15절 closed environment, ADR-0012 결정 6과 정합).

## 3. 근거

### 3.1 캡션 추가 (결정 1)
- **법적 안전.** 표지 노출은 저작물 사용이다. 제목·저자를 함께 보이면 4요소 중 2요소가 화면에 직접 표시되어 어트리뷰션 충족도가 크게 올라간다.
- **hover 툴팁은 부적합.** 후보였던 "표지 hover 시 어트리뷰션 툴팁"은 키키북스의 주 기기인 모바일(390px)에 hover가 없어, 정작 가장 많이 쓰는 기기에서 어트리뷰션이 표시되지 않는다.
- **푸터 안내만으로는 부족.** 비로그인 방문자는 표지를 누르면 `/signup`으로 가므로 책 상세의 `AttributionBox`에 도달할 수 없다. "상세 페이지에서 확인" 안내만 두면 그 경로가 비로그인 상태에서 공허해진다. 화면에 제목·저자가 직접 있어야 한다.
- **UX 개선.** 표지만 있고 제목이 없으면 학부모가 무슨 책인지 알 수 없다. 캡션은 어트리뷰션이자 UX다.

### 3.2 두 줄→제목만 후퇴 여지 (결정 1)
- 모바일 390px에서 6권 그리드는 2열이며 표지 셀이 좁다. 제목·저자 두 줄이 과밀하면 가독성이 떨어진다. 디자인을 코드로 확정하기 전 실제 레이아웃을 보고 판단하는 것이 정직하다(claude.md 1절 — 추측 금지).
- "제목만"으로 후퇴해도 결정 2 안내가 라이선스·저작자 도달 경로를 보장하므로 법적 하한선은 지켜진다.

### 3.3 화면별 정책 단일화 (결정 1~4)
- phase-10·11·13이 책 목록을 반복해서 만든다. 화면마다 어트리뷰션을 즉흥 결정하면 누락·불일치가 생긴다. 본 ADR을 정본으로 두고 후속 화면이 모두 참조하면 일관성이 강제된다.

## 4. 결과

- phase-09a 인기 책 표지 카드는 표지 + 제목·저자 캡션으로 구성된다. `lib/landing/popular-books.ts`의 조회 컬럼에 `title`·`author`를 포함한다.
- phase-09a 랜딩 푸터에 결정 2 안내 문구가 들어간다. 문구 자체는 `lib/landing/copy.ts`가 보유한다(ADR-0012 결정 2).
- phase-10(홈)·phase-13(라이브러리)은 책 목록 표지 노출 시 본 ADR을 참조해 같은 캡션·안내 정책을 적용한다.
- phase-11 책 상세의 `AttributionBox`(4요소 전체) 의무는 `license-rules.md` 5절 그대로 — 본 ADR은 그 의무에 손대지 않는다.
- DB 스키마 변경 없음. `books`의 `title`·`author`는 기존 컬럼이다.

## 5. 미반영 항목 (의도적 보류)

- **그림 작가(illustrator) 캡션 노출** — 목록 캡션은 글 저자(author)까지만. illustrator 포함 4요소 전체는 책 상세 `AttributionBox`가 담당.
- **표지 위 라이선스 배지·워터마크** — 베타에서는 캡션 + 안내 문구로 충분. 필요 시 후속 검토.
- **caption 다국어** — 베타는 한국어 단일.

---

## 6. 재검토 트리거

- **phase-09a CP3 구현** — "제목 + 저자 두 줄"이 모바일에서 과밀한지 실측 후, "제목만" 후퇴 여부를 결정 1에 따라 확정·보고.
- **phase-10·13 책 목록 작업** — 본 ADR 정책을 그대로 적용하되, 카드 크기가 랜딩과 크게 다르면 캡션 범위를 재검토.
- **법무 검토(phase-14)** — 변호사 검토에서 목록 화면 어트리뷰션 표시 수준에 대한 의견이 나오면 결정 1·2를 갱신.

## 7. 후속 과제 — illustrator 데이터 부재 (phase-11 진단 대상)

⚠️ phase-09a CP2 실측에서 **활성 책 896권 전체(896/896, 100%)가 `illustrator`가 NULL**임을 확인했다.

`license-rules.md` 5.1절과 `PLAN.md` 9절(Week 6)은 책 상세 페이지 `AttributionBox`에 "🎨 그림: {illustrator}" 행을 명시한다. 그러나 현재 데이터로는 이 행을 채울 수 없다.

- 본 ADR(목록 화면 캡션)은 illustrator를 캡션에 넣지 않으므로(결정 1·§5) phase-09a에는 영향이 없다.
- **phase-11(Screen 03 책 상세) 진입 시** 다음을 진단하고 **별도 ADR**로 다룬다:
  1. phase-04·05 동기화 스크립트(`scripts/sync_book_dash.py`·`scripts/sync_gdl.py`)가 illustrator 필드를 적재하지 않는 것인지, 원천(Book Dash `meta.yml`·GDL API)에 필드 자체가 없는 것인지 진단.
  2. `PLAN.md` 9절 "🎨 그림: {illustrator}" 명세를 데이터 부재 실태에 맞춰 재설계(행 생략·조건부 표시 등).
- 이 진단·재설계는 phase-11 범위다. 본 ADR은 그 트리거를 박제하는 데서 그친다.
