# ADR-0050: 회전 페이지 처리 — 자동교정 없음·원본 그대로 렌더

## Status
Accepted (2026-07-16) / 기준 HEAD d186019

## Context

- 검수 대상 텍스트 중 일부 페이지가 **읽기순서 역전 상태**다(예: `catch-that-cat` p5 `ssendoog knahT` = "Thank goodness" 역순). 원인은 추출 실패가 아니라 **원본 페이지의 텍스트가 90° 회전 인쇄**되어 있고 `order_fix`가 가로 기준으로 정렬했기 때문이다 — 팀장 육안 확인(2026-07-16, `out_images_154/catch-that-cat/05.jpg`).
- 전수 실측(`scripts/pdf_harvest/audit_rotation.py`, `scratchpad/rotation_audit_154.csv`):
  - 본문 2,156면 중 **ROTATED 81면(3.8%)**
  - = 직교회전 (0,±1)/(−1,0) **33면 / 18권** + 대각선만 (1,±1) **48면**
  - `page.rotation`(/Rotate)은 2,156면 **전면 0**.

## Decision

### D1. 회전 페이지의 텍스트 자동 교정기를 만들지 않는다
- 근거: 직교회전은 2,156면 중 **33면(1.5%)**이다. 교정기 개발·정답지 구성·검증 비용이 33면을 검수 화면에서 수정하는 비용을 상회한다. 33면은 Phase C 검수의 정상 업무 범위 안이다.

### D2. 페이지 이미지는 원본 그대로 렌더한다(회전 보정 없음)
- 근거: `page.rotation`이 전면 0이므로 PDF 상 페이지 자체는 회전돼 있지 않다. 회전은 원본 저작물의 편집 의도다 — 세로로 긴 장면을 가로 판형에 배치한 결과이며(팀장 확인), 이미지를 돌리면 원본과 달라진다. 검수자는 원본과 동일한 화면을 봐야 한다.

### D3. 회전 페이지 목록을 검수 우선순위 자료로 보존한다
- `scratchpad/rotation_audit_154.csv`(`is_rotated=1`)는 검수 화면에서 "주의 페이지" 표시에 활용 가능하다. 다만 이번 트랙에서 화면에 반영하지는 않는다(Phase C 설계 시 판단).

### D4. 대각선 텍스트 (1,±1) 48면은 교정·표시 대상이 아니다
- 근거: 아크형 SFX·장식 텍스트로 원본 디자인이다. 페이지는 정상 방향이다.

## Consequences

- **얻는 것**: 트랙 단순화. 이미지가 원본과 동일. 회전 목록이 검수 자료로 남음.
- **잃는 것**: 33면의 텍스트 순서를 사람이 수정해야 한다.
- **되돌리기**: 회전 교정이 필요해지면 `order_fix`에 dir 기반 정렬을 추가하고 해당 33면만 재생성하면 된다(`book_text` UPDATE, 스키마 무변경).

## Non-goals

- `order_fix` 재설계 / 검수 화면 UI / 이미지 업로드 규약(ADR-0049).
