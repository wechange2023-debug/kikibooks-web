# ADR-0010 — 이메일 confirmation 메일 발송 보류 (베타 직전 커스텀 SMTP 연결로 이관)

**상태** Accepted
**날짜** 2026-05-18
**관련** `docs/adr/0009-auth-architecture.md`(§3.1 인증 공급자 범위), `tasks/phase-07-auth.json`(v6 검증 항목), `docs/intent/auth-flow.md`(§4.1 이메일 회원가입 흐름), `PLAN.md` 8절(Phase 0)·9절(Phase 1)

---

## 1. 배경

phase-07(인증) 클릭 테스트 v6(이메일 회원가입 → 확인 메일 → 로그인)을 검증하던 중, Supabase 프로젝트의 **기본 내장 SMTP**로는 confirmation 메일이 도착하지 않는 현상을 확인했다(2026-05-18).

- Supabase Auth Logs에는 `/signup` request completed가 정상 기록됨 — 키키북스 코드 경로(회원가입 폼 → 서버 액션 → `signUp` 호출 → "메일함을 확인해 주세요" 안내 화면)는 모두 정상 동작.
- 미도착의 원인은 Supabase가 기본 제공하는 공유 SMTP의 알려진 제약(엄격한 시간당 발송 한도·낮은 도달률·지연)에 있다. 이는 프로덕션 사용 시 **커스텀 SMTP 연결**을 전제로 한 설계다.

즉 v6의 미완 지점은 키키북스 애플리케이션 코드가 아니라 외부 메일 인프라 설정에 있다.

---

## 2. 결정

**이메일 가입 confirmation 메일의 실제 발송·도달 검증은 phase-07에서 완결하지 않고, 베타 출시 직전 별도 단계에서 커스텀 SMTP 서비스(예: Resend) 연결로 해결한다.**

- phase-07은 인증 코드 경로(이메일·Google·세션·보호 라우트·프로필 생성)가 검증된 상태로 **완료 처리**한다.
- v6의 "메일 발송" 부분만 **보류(deferred)** 상태로 `tasks/phase-07-auth.json` v6 항목에 명시 기록한다.

---

## 3. 근거

### 3.1 phase-07 범위 밖의 문제
- `tasks/phase-07-auth.json`의 `scope_in`은 "Supabase Auth 이메일 공급자 연결"이며, SMTP 인프라 선정·연결은 범위에 없다.
- 키키북스 코드(폼·서버 액션·콜백·프로필 upsert)는 v6에서 정상 동작을 확인했다 — 코드 결함이 아니다.

### 3.2 Google 로그인이 정상 작동 → 베타 진행에 지장 없음
- v7(Google OAuth)은 완전히 통과했다(`/home` 도착·`profiles` 행 생성 확인).
- 베타 테스터는 Google 로그인만으로도 모집·온보딩이 가능하다(ADR-0009 §3.1 — 베타 테스터 대부분 Google 계정 보유).
- 따라서 이메일 메일 발송 미완은 베타 일정의 **블로커가 아니다.**

### 3.3 커스텀 SMTP는 어차피 출시 전 필수 작업
- 비밀번호 재설정·알림 등 향후 메일 기능도 커스텀 SMTP를 전제로 한다(ADR-0009 §5).
- phase-07 시점에 SMTP를 급히 붙이기보다, 출시 직전 메일 인프라 단계에서 한 번에 연결·검증하는 편이 응집도가 높다.

---

## 4. 결과

- phase-07은 v1~v5(lint·type-check·build·스키마 무변경·secret 미노출)와 v7~v11(Google·보호 라우트·프로필·로그아웃) 통과로 **완료**된다.
- v6은 "코드 경로 정상 / 메일 발송 보류"로 `tasks/phase-07-auth.json`에 기록된다.
- `docs/intent/auth-flow.md` §4.1의 이메일 회원가입 흐름(확인 메일 단계)은 유효하나, 실제 메일 도달은 SMTP 연결 후에 보장된다.
- 이메일 가입 자체를 막지는 않는다 — Supabase 설정에서 "Confirm email"을 끄면 메일 없이도 가입이 가능하나, 베타 보안상 켜둔 상태를 유지하고 SMTP 연결을 기다린다.

---

## 5. 미반영 항목 (의도적 보류)

- **커스텀 SMTP 서비스 선정·연결**(Resend 등) — 베타 출시 직전 별도 phase.
- **비밀번호 재설정 메일** — 같은 SMTP 인프라에 의존, ADR-0009 §5와 함께 베타 후.
- **메일 템플릿(브랜딩) 커스터마이즈** — SMTP 연결 단계에서 함께 처리.

---

## 6. 재검토 트리거

- **베타 출시 직전 SMTP 연결 phase 진입** — 본 ADR이 직접 다루는 후속 작업. 이때 v6의 메일 발송·도달을 재검증한다.
- SMTP 연결 전에 이메일 가입 사용자 비중을 늘려야 할 사업적 필요가 생기면 앞당겨 검토.
