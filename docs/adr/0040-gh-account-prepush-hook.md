# ADR-0040: gh 계정 드리프트 차단 — pre-push 훅

- 상태: Accepted
- 날짜: 2026-07-13
- 관련: ADR-0020(git 규율), 인계메모 2026-07-10 안건 #1

## 맥락
push는 반드시 crspiegel 계정으로만 해야 한다(레포 권한 구조상 타 계정은 403).
그러나 팀장이 2개 VS Code·다중 GitHub 계정을 동시 운영하며, 활성 gh 계정이
crspiegel 아닌 값으로 바뀌는 드리프트가 반복 발생했다. 2026-07-10 세션에서만
5회 재발. 이는 조작 실수가 아니라 환경 결함으로 분류한다.

## 결정
1. git pre-push 훅으로 push 직전 활성 gh 계정을 검사한다.
2. 판별은 로컬 명령 `gh auth status --active`를 1차로 사용한다.
   (오프라인에서도 작동해야 하므로 네트워크 호출 gh api user는 쓰지 않는다.)
3. 활성 계정이 crspiegel이 아니면 push를 차단하고(exit 1) 멈춘다.
   — fail-closed. 훅은 자동으로 계정을 전환하지 않는다.
   — 이유: "crspiegel 아니면 즉시 멈추고 보고"가 불변 규율이다. 자동 전환은
     팀장이 사고를 인지할 기회를 없앤다. 문지기는 막고 알린다.
4. 훅은 scripts/git_hooks/ 아래에 두고 core.hooksPath로 연결한다.
   (.git/hooks는 clone 시 재현 안 됨 → 버전관리되는 scripts/ 아래에 둔다.)

## 결과
- 장점: crspiegel 아닌 계정 push가 물리적으로 차단됨. 오프라인에서도 작동.
- 비용: gh 미설치·keyring 접근 불가 등 예외 시 훅이 push를 막을 수 있음
  → 그 경우 진단 메시지를 명확히 출력해 팀장이 원인을 알게 한다.
- 우회: 훅 자체 문제로 정당한 push가 막히면 --no-verify로 우회 가능하나,
  이는 예외 상황 기록을 남긴 뒤에만 쓴다.
